Design Lab

20 Time Input Patterns

Styled to sit next to the existing Anchor admin editors instead of fighting them. All smart typing examples below accept shorthand like 1300, 930, 17, or 1pm and normalize on blur.

Top pick to test: Hybrid row Typing-friendly: Smart single field Safest for staff: Quarter-hour dropdown Good compromise: Smart range pair

Normalization

Examples: 1300 -> 1:00 PM, 0830 -> 8:30 AM, 17 -> 5:00 PM.

Style Goal

No white browser-default blocks. Every prototype uses the same editor surfaces, border radius, focus ring, and button language as the rest of the site.

Decision Lens

We are comparing typing speed, readability, mobile fit, overnight clarity, and how hard each pattern is to break in day-to-day admin use.

1

Smart Single Field

Typing

Fastest free-form option. Good if staff already think in shorthand.

2

Smart 24-Hour Field

Kitchen style

Useful if the editor stores 24-hour values but still previews 12-hour meaning.

3

Input With Live Suffix

Compact

Keeps the field narrow while confirming the interpreted period.

4

Smart Range Pair

Strong candidate

Closest to the current editor, but with shorthand-friendly entry.

Why it works: familiar open/close structure, consistent styling, and no browser-native white controls.
5

Datalist Smart Field

Suggestive

Typing stays available, but staff also get visible suggestions.

6

Smart Field With Quick Chips

Fast edits

Lets staff type anything, but common values remain one click away.

7

Quarter-Hour Dropdown

Safest

High reliability. Great when you want to prevent malformed entry entirely.

8

Half-Hour Dropdown

Less noise

Cleaner than 15-minute lists if the business rarely uses odd times.

9

Dropdown Range Pair

Verbose

Very safe, but can feel heavy if editors change times often.

10

Split Hour / Minute / AM-PM

Explicit

Hard to misunderstand, but it adds more moving parts to the row.

11

Closed Toggle + Hours

Practical

Makes “closed” a first-class choice instead of an implied empty state.

12

Overnight Row

Drinks-friendly

Keeps the next-day concept visible when a shift runs past midnight.

13

Closed / All Day / Timed

State first

Great when the restaurant frequently marks tabs closed or all-day.

14

Start Time + Duration

Less math

Staff chooses when a tab starts and how long it runs.

15

Open Time + Close After

Operational

Feels natural when meal periods are usually predictable blocks.

Smart Combo Box Row

Best overall
